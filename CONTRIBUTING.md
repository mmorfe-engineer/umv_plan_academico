# Contribuyendo a Academic Tracker UVM

¡Primero que nada, gracias por tu interés en contribuir! 🎉

Este documento describe cómo puedes contribuir al proyecto **Academic Tracker UVM**.

---

## 📋 Cómo Contribuir

### 1. Reportando Bugs

Si encuentras un bug, por favor abre un **Issue** en GitHub con la siguiente información:

- **Descripción clara** del problema
- **Pasos para reproducir** el error
- **Comportamiento esperado** vs **Comportamiento actual**
- **Capturas de pantalla** (si aplica)
- **Versión de Python** y sistema operativo

### 2. Sugiriendo Mejoras

Las ideas para nuevas características son bienvenidas. Abre un **Issue** con:

- Descripción detallada de la característica
- Casos de uso
- Beneficios para el proyecto

### 3. Contribuyendo con Código

#### Requisitos Previos
- Git
- Python 3.11+
- Conocimientos básicos de GitHub

#### Pasos

1. **Fork el repositorio**
   ```bash
   git fork https://github.com/mmorfe-engineer/umv_plan_academico
   ```

2. **Clona tu fork**
   ```bash
   git clone https://github.com/tu-usuario/umv_plan_academico.git
   cd umv_plan_academico
   ```

3. **Crea una nueva branch**
   ```bash
   git checkout -b feature/nueva-caracteristica
   ```

4. **Haz tus cambios**
   - Sigue el estilo de código existente
   - Añade pruebas si aplica
   - Actualiza la documentación

5. **Haz commit de tus cambios**
   ```bash
   git add .
   git commit -m "feat: añade nueva característica"
   ```

6. **Empuja a tu fork**
   ```bash
   git push origin feature/nueva-caracteristica
   ```

7. **Abre un Pull Request**
   - Ve a [Pull Requests](https://github.com/mmorfe-engineer/umv_plan_academico/pulls)
   - Haz clic en "New Pull Request"
   - Selecciona tu branch
   - Proporciona una descripción clara de tus cambios

---

## 📝 Estándares de Código

### Python
- Sigue **PEP 8** (Guía de estilo de Python)
- Usa nombres descriptivos para variables y funciones
- Añade comentarios para código complejo
- Docstrings para funciones y clases

### Git
- Mensajes de commit **claros y descriptivos**
- Usa prefijos:
  - `feat:` para nuevas características
  - `fix:` para correcciones de bugs
  - `docs:` para cambios en documentación
  - `chore:` para tareas de mantenimiento

### Ejemplo de Commit
```bash
feat: añade soporte para notificaciones por Telegram
docs: actualiza README con nueva sección de instalación
fix: corrige filtrado de fechas en Weekly Monday Review
```

## 🚀 Revisión de Pull Requests

Todos los Pull Requests serán revisados por los mantenedores del proyecto.

**Criterios de aceptación:**
- ✅ Código funciona correctamente
- ✅ No rompe funcionalidad existente
- ✅ Sigue los estándares de código
- ✅ Incluye documentación actualizada
- ✅ Tiene pruebas (si aplica)
